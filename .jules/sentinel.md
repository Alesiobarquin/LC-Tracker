## 2024-05-03 - Prevent XSS in Image src and href Attributes
**Vulnerability:** XSS (Cross-Site Scripting) via `javascript:` URI injection in user-supplied `image_url` fields within `src/components/AdminDashboard.tsx`.
**Learning:** While the `href` attribute on the wrapping `<a>` tag had an inline `try/catch` to check the URL protocol, the `src` attribute on the nested `<img>` tag directly rendered the unsanitized user input (`ticket.image_url`). This allowed a malicious user to craft an `image_url` containing `javascript:alert(1)` which could execute scripts if parsed by older browsers or if the image failed to load and was interacted with, bypassing the `href` validation.
**Prevention:** Always sanitize user-provided URLs before using them in ANY HTML attribute (`href`, `src`, `action`, etc.). Use a centralized utility function (like `sanitizeUrl` in `src/utils/security.ts`) to strictly enforce allowed protocols (e.g., `http:` and `https:`) rather than relying on ad-hoc inline validation blocks.
